## Known Patterns

- **Unmerged branch pattern**: Tasks are regularly completed in local worktrees (ao/task-XXX) and marked "done" but never pushed to origin or merged to main. This has recurred every single run. TASK-010/018/021/026/035/047 were all created to address this. The fix (TASK-047) remains in backlog.
- **pnpm db:push not run after merges**: Every time schema-changing branches merge (TASK-022, TASK-031, TASK-033), the DB isn't updated — causing SqliteError crashes on dashboard/posts/calendar/analytics/campaigns. Pattern: merge → crash → TASK to run db:push.
- **AI gen always 500/401**: ANTHROPIC_API_KEY is set in .env but invalid — Anthropic API returns 401 authentication_error. As of run 10, raw 401 JSON is exposed directly to user in the UI.
- **Empty "done" branches**: Tasks are marked "done" but the ao/task-XXX branch contains ZERO unique commits (branch just points to a recent main commit). Seen with TASK-045/046/047/051/052/053/054 in runs 10-11. The worktrees exist but no work was committed. Different from the "unmerged branch" pattern — here no work was done at all. Now 11 consecutive runs with this pattern.
- **Dev server environment churn**: PostPilot dev server startup is not stable across runs. Previously CondoHub occupied :3000 (runs 1-15), PostPilot ran on :3001. Run 16: launchapp-nextjs occupied :3000/:3001/:3002, PostPilot not running. Run 17: environment settled — :3000=CondoHub, :3001=PostPilot (PID 65539), :3002=Invoicer. BUT PostPilot server hangs after BUG-015 crash on initial request — accepts TCP but no HTTP responses.
- **BUG-015 causes server hang**: When PostPilot dev server starts and receives first request to `/`, it logs the buttonVariants() error and then stops responding to ALL HTTP requests (TCP connections still accepted). This means even auth/api routes are unreachable after the server hits BUG-015. Fixing BUG-015 is now doubly critical — it not only breaks the landing page but renders the entire dev server non-functional.
- **Ghost-done BUG-015 fix (21+ times)**: TASK-046 through TASK-090 (21+ tasks) all marked "done" for deleting button.tsx. button.tsx still exists on main (only ever had commit 63cd453). No deletion commit has EVER been made. button.ts (the correct barrel, no "use client") exists alongside button.tsx. TASK-091 also ghost-done for accounts pages (same pattern — zero unique commits).
